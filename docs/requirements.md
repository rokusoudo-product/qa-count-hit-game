# 要件定義書

**プロジェクト名**: 人数当てゲーム
**作成日**: 2026-03-20
**バージョン**: 1.0
**作成者**: POエージェント（株式会社六創堂）

---

## 1. プロダクトビジョン

複数人がリアルタイムでQRコードを使ってグループを形成し、質問への回答人数を予測し合う、
パーティー向けAndroidアプリ。六創堂の技術力アピールとしてGoogle Playで公開する。

---

## 2. ステークホルダー

| 役割 | 説明 |
|------|------|
| ホスト | ルームを作成・進行管理する人 |
| 参加者 | QRコードでルームに参加する人 |
| 株式会社六創堂 | プロダクトオーナー・公開主体 |

---

## 3. 機能要件

### 3.1 ルーム管理
- ホストがルームを作成できる
- ルーム作成時にQRコードが生成される
- QRコードをスキャンしてルームに参加できる
- ルームには最大20人まで参加できる（初期設定）
- ホストがゲームの開始・終了を制御できる

### 3.2 質問・回答フェーズ
- ホストが事前設定した質問が全参加者に表示される
- 質問は二択形式（例：「犬派？猫派？」）
- 全員が時間内に回答する（タイムアウト設定あり）
- 回答はリアルタイムでサーバーに送信される

### 3.3 予測フェーズ
- 回答集計後、「〇〇を選んだのは何人か？」という予測フェーズに移行
- 各参加者が人数を入力して予測する
- タイムアウト設定あり

### 3.4 採点・結果表示
- 予測値と実際の回答人数の差が小さいほど高得点
- ラウンドごとのスコアと累計スコアを表示
- 最終的なランキングを表示

### 3.5 リアルタイム通信
- WebSocket（AWS API Gateway）を使ったリアルタイム同期
- 参加者の入退室・回答状況をリアルタイム反映

---

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| プラットフォーム | Android（minSdk 26 / API Level 26以上） |
| 同時接続数 | 1ルームあたり最大20人 |
| レスポンス | 回答反映は3秒以内 |
| 可用性 | AWS Lambda + DynamoDB（サーバーレス構成） |
| セキュリティ | ルームIDはUUID、外部から推測不可 |
| コスト | AWS無料枠での運用を優先 |

---

## 5. スコープ外（v1.0）

- iOSアプリ対応
- アカウント登録・ログイン機能
- カスタム質問作成（初期はプリセット質問のみ）
- 音声・動画チャット
- 広告掲載

---

## 6. 制約事項

- Google Playポリシー準拠
- 個人情報は収集しない（ニックネームのみ）
- AWS東京リージョン使用
